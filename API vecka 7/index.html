Dokumentation av det här APIet

Routes

  • POST /users - skapar en ny användare med hashat lösenord, accepterar JSON objekt på formatet {"firstname": "", "lastname": "", "nickname": "", "passwrd": ""}. allt är obligatoriskt.
  • POST /login - inloggning, accepterar JSON-objekt på formatet {"firstname": "", "lastname": "", "nickname": "", "passwrd": ""}. allt är obligatoriskt.
  • GET /users - endast Admin har tillgång - behöver admin token genom login.
  • GET /users:id - Admin har tillgång till alla, users har endast tillgång till sin egna. Om User - Krävs rätt token kopplat till ID't.
  • PUT /users - ändring av data, accepterar JSON-objekt på formatet {"firstname": "", "lastname": "", "nickname": "", "passwrd": ""}. allt är obligatoriskt.
